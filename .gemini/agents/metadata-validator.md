# Metadata Validator

마크다운 파일의 메타데이터 완전성을 검사하고 보고서를 생성하는 에이전트입니다.

기록학에서 메타데이터는 기록물의 맥락을 보존하는 핵심 요소입니다. 이 에이전트는 파일에 제목과 날짜가 제대로 기록되어 있는지 자동으로 검사합니다.

## 역할

- 파일의 메타데이터(제목, 날짜) 존재 여부 확인
- 파일명 규칙(YYYY-MM-DD) 준수 여부 확인
- 검사 결과 보고서 생성

## 프로세스

### Step 1: 파일 찾기

Glob으로 검사 대상 파일을 찾는다.

```
Glob: 20-created/**/*.md
Glob: 30-collected/**/*.md
```

제외 대상:
- `README.md`
- `.gitkeep`

### Step 2: 각 파일 검사

Read로 각 파일을 열어 다음 항목을 확인한다.

| 검사 항목 | 확인 방법 | 통과 기준 |
|----------|----------|----------|
| 날짜 형식 | 파일명이 `YYYY-MM-DD`로 시작하는가? | 파일명 앞 10자리가 날짜 형식 |
| 제목 | 첫 번째 `# 제목`이 있는가? | `#`으로 시작하는 줄 존재 |
| frontmatter | YAML frontmatter(`---`)가 있는가? | 파일 첫 줄이 `---` |

각 항목은 통과(O) 또는 미통과(X)로 기록한다.

### Step 3: 결과 정리

검사 결과를 표로 정리한다.

```markdown
| 파일명 | 날짜 형식 | 제목 | frontmatter | 결과 |
|--------|----------|------|-------------|------|
| 2026-04-01-기록학-개론.md | O | O | O | 통과 |
| 메모.md | X | O | X | 미통과 |
| 2026-03-30-웹클립.md | O | X | X | 미통과 |
```

### Step 4: 보고서 저장

Write로 검사 결과를 저장한다.

- 저장 위치: `20-created/YYYY-MM-DD-metadata-validation-report.md`
- 파일명의 날짜는 오늘 날짜를 사용

보고서 형식:

```markdown
---
title: 메타데이터 검증 보고서
date_created: YYYY-MM-DD
---

# 메타데이터 검증 보고서

검사 일시: YYYY-MM-DD

## 검사 결과

| 파일명 | 날짜 형식 | 제목 | frontmatter | 결과 |
|--------|----------|------|-------------|------|
| ... | ... | ... | ... | ... |

## 요약

- 검사 파일: N개
- 통과: N개
- 미통과: N개
- 통과율: N%

## 미통과 파일 상세

### 파일명: 메모.md
- 날짜 형식: X (파일명에 YYYY-MM-DD 없음)
- frontmatter: X (YAML frontmatter 없음)
- 권장: 파일명을 `YYYY-MM-DD-메모.md`로 변경하고 frontmatter 추가
```

## 주의사항

- 파일을 수정하지 않는다. 검사만 하고 보고서를 생성한다.
- 빈 폴더는 건너뛴다.
- 보고서에 검사 일시를 반드시 기록한다.
- 보고서 파일명도 YYYY-MM-DD 규칙을 준수한다.

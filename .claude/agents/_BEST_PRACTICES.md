# 에이전트 만드는 법

이 문서는 `.claude/agents/` 폴더에 에이전트를 만들 때 참고하는 가이드입니다.

---

## 에이전트란?

에이전트는 AI에게 특정 역할과 작업 절차를 미리 정의해놓은 문서입니다.
프롬프트를 매번 처음부터 쓰는 대신, 자주 하는 작업을 에이전트로 만들어두면 한 마디로 실행할 수 있습니다.

예시:
- "메타데이터 검사해줘" -> `metadata-validator` 에이전트가 자동 실행
- "이 노트에 관련 노트 연결해줘" -> `zettelkasten-linker` 에이전트가 자동 실행

---

## 에이전트 파일 구조

모든 에이전트는 `.claude/agents/` 폴더에 `.md` 파일로 저장합니다.

### 필수 구성요소

```markdown
---
name: 에이전트-이름
description: 에이전트가 하는 일을 한 줄로 설명. 트리거 키워드 포함.
tools: Read, Glob, Grep
model: haiku
---

작성일자: YYYY-MM-DD
최종수정일자: YYYY-MM-DD
수정이력:
- YYYY-MM-DD: 초기 작성

# 에이전트 이름

에이전트의 역할을 설명합니다.

## 프로세스

### Step 1: 파일 찾기
Glob으로 대상 파일을 찾는다.

### Step 2: 파일 읽기
Read로 각 파일을 읽는다.

### Step 3: 결과 정리
분석 결과를 정리한다.
```

### YAML 필드 설명

| 필드 | 필수 | 설명 |
|------|------|------|
| `name` | O | 에이전트 이름 (영문, 하이픈 사용) |
| `description` | O | 에이전트 설명 + 자동 실행 트리거 키워드 |
| `tools` | O | 에이전트가 사용할 수 있는 도구 목록 |
| `model` | O | AI 모델 선택 |

---

## 사용 가능한 도구 (Tools)

| 도구 | 하는 일 | 권한 |
|------|---------|------|
| **Read** | 파일 내용을 읽는다 | Level 1+ |
| **Glob** | 패턴으로 파일을 찾는다 (예: `20-created/**/*.md`) | Level 1+ |
| **Grep** | 파일 내용에서 키워드를 검색한다 | Level 1+ |
| **Write** | 새 파일을 만든다 | Level 2+ |
| **Edit** | 기존 파일을 수정한다 | Level 3+ |
| **Bash** | 터미널 명령을 실행한다 | Level 2+ (제한적) |
| **WebSearch** | 웹에서 정보를 검색한다 | Level 3+ |

에이전트에는 필요한 도구만 넣으세요.
읽기만 하는 에이전트에 Write를 넣을 필요가 없습니다.

---

## 모델 선택

| 모델 | 특징 | 추천 용도 |
|------|------|----------|
| `haiku` | 빠르고 가벼움 | 단순 검사, 파일 목록, 통계 |
| `sonnet` | 균형잡힌 성능 | 분석, 요약, 노트 작성 |
| `opus` | 가장 똑똑함 | 복잡한 판단, 긴 문서 분석 |

---

## 핵심 원칙

### 하나의 에이전트, 하나의 역할

- 나쁜 예: "파일 검사 + 요약 + 번역 + 정리" (한 에이전트에 너무 많은 역할)
- 좋은 예: "파일 메타데이터 검사" (하나의 명확한 역할)

### 날짜 추적 필수

모든 에이전트 파일에 작성일자, 최종수정일자, 수정이력을 기록합니다.
에이전트를 수정하면 최종수정일자를 갱신하고 수정이력에 추가합니다.

### 단계별로 작성

프로세스를 Step 1, Step 2, Step 3... 으로 명확하게 나누세요.
각 단계에서 어떤 도구를 쓰는지 적으면 AI가 정확하게 실행합니다.

---

## 포함된 예시 에이전트

| 에이전트 | 권한 레벨 | 복잡도 | 설명 |
|---------|----------|--------|------|
| `folder-analyzer` | Level 1 | 가장 단순 | 폴더 구조 분석, 채팅으로 출력 |
| `metadata-validator` | Level 2 | 단순 | 메타데이터 검사 + 보고서 생성 |
| `concept-learning-partner` | Level 2 | 보통 | 대화형 개념 학습 + 노트 저장 |
| `zettelkasten-linker` | Level 3 | 보통 | 노트 간 자동 연결 |

자신만의 에이전트를 만들 때, `metadata-validator`를 참고하면 가장 쉽습니다.
4단계 (찾기 -> 읽기 -> 검증 -> 저장)로 구성된 가장 기본적인 패턴입니다.
